# 1. Codex Mission

Open and work only in the existing **TRUSTed Digital Architecture** website repository.

Before changing code, read:

1. `PROJECT_STATUS.md`
2. `BACKLOG.md`
3. `DECISIONS.md`
4. `README.md`
5. the current homepage implementation
6. all existing route files, shared components, navigation, footer, and styles
7. this document in full

Treat this document as the founder-approved content and structural specification for the public marketing website.

Do not create a separate application.  
Do not add authentication, a database, payments, external AI providers, production API keys, or backend integrations during this mission.  
Build the public-facing website architecture, page copy, visual system, responsive layouts, and clearly labeled non-functional demonstrations or intake interfaces where appropriate.

The current website visual direction is approved:

- quiet luxury
- premium editorial typography
- warm ivory backgrounds
- deep architectural green
- restrained gold accents
- precise grid systems
- subtle technical diagrams
- generous spacing
- minimal visual noise
- strong contrast
- polished micro-interactions
- modern business architecture imagery

Preserve and refine the existing visual language. Do not redesign it into a generic SaaS template.

# 31. Visual and UX Requirements

Preserve the approved visual language from the current design.

## Typography

- high-contrast editorial serif for major headlines
- refined sans serif for body copy, navigation, controls, labels, and technical details
- avoid oversized headline wrapping that becomes difficult to scan on mobile

## Color

- warm ivory or parchment
- deep architectural green
- restrained muted gold
- neutral gray-green text
- no bright gradients
- no generic purple SaaS styling
- no excessive glowing effects

## Layout

- strong grid
- editorial asymmetry
- large whitespace
- technical diagrams
- thin architectural rules
- consistent section rhythm
- premium card borders
- clear responsive stacking
- no overcrowded icon walls

## Motion

- subtle
- purposeful
- reduced-motion compatible
- no animation that delays reading
- architecture diagrams may animate lightly on intersection
- hover states should remain restrained

## Accessibility

- semantic headings
- keyboard navigation
- visible focus states
- sufficient contrast
- meaningful link labels
- form labels and errors
- reduced-motion support
- appropriate ARIA only where needed
- responsive zoom and reflow

- can support

- subject to technical discovery
